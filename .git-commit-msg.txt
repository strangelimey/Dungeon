Threads: add a Core thread Manager and port the AI onto it (step 1)

First slice of the engine-wide thread subsystem: a central registry of
managed worker threads (Core/ThreadManager.h, namespace dungeon::threads)
that everything threaded becomes a CLIENT of, so control is uniform.

threads::Manager:
- Spawn(job, {name, hz}) -> a named worker that runs the job once per tick in
  a loop the manager owns, at the given cadence (hz=0 = flat-out).
- Cooperative cancellation: each worker gets a std::jthread stop_token; the
  cadence sleep is a condition_variable_any that wakes the instant a stop is
  requested. RequestStop is non-blocking; Stop requests + joins (clients call
  it in their dtor before captured state dies).
- Lock-free Inspect / SnapshotAll: per-worker stats are atomics (state,
  iterations, last/avg/max tick ms, heartbeat age, last error), so reading a
  worker never blocks it.
- Per-tick crash capture: a throwing job records the error and the worker
  keeps running instead of std::terminate taking down the process.
- OS thread naming via SetThreadDescription so workers show by name in the
  debugger / profilers.

The Manager is owned by Game (declared before m_world so it outlives every
client) and passed down through DungeonWorld to the AI.

AI port: ai::AsyncDirector no longer owns raw std::threads. It Spawns one
named worker per IQ bucket ("ai.bucket0".."ai.bucket3") at the bucket cadence
via the Manager; the old per-bucket loop body becomes ComputeBucket, the unit
of work the manager ticks. Its dtor Stop()s those workers before its snapshot/
plan state dies. Snapshot/plan handoff is unchanged.

Verified: builds clean; a 32x32 / 13-monster save loads and runs with the
four bucket workers live, no asserts.

Pinning the Manager ctor + dtor out-of-line keeps the opaque Worker type
(unique_ptr<Worker> members) from leaking into client TUs.

Next steps on this foundation: live throttle (pause / SetRate / global
governor), watchdog/stall detection, a `threads` dev-console inspector, then
priority/affinity + the quarantine Kill layer.
