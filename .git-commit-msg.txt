Threads: live per-thread control in the dev console + Manager throttle

Add the control half of the thread subsystem and surface every worker in
the dev console's top panel, where the perf indicators live.

threads::Manager controls (immediate — they wake the worker's cadence sleep,
flipping the atomics under the sleep mutex then notifying so a sleeper can't
miss the wake):
- Pause / Resume (new State::Paused; the worker holds without joining and runs
  no job until resumed or stopped).
- SetRate(hz) — change a worker's re-run cadence on the fly (wakeGen bump makes
  the in-flight sleep recompute its interval).
- WorkerInfo gains `paused` so the UI reflects intent immediately.

DevConsole:
- Takes a threads::Manager& (wired from Game, which owns it).
- A THREADS section under the perf block lists every managed worker live:
  name, state (colored — paused yellow, dead dim), iterations, last/avg tick
  ms, and hz. Each row has four clickable buttons: halt/run (pause/resume),
  << / >> (halve / double rate, clamped 0.1..60 hz), and kill (RequestStop).
- Render lays out the button rects and records them in m_threadHits; the next
  Update hit-tests left-clicks against them (gfx::Rect::Contains), so the
  layout lives in exactly one place and the one-frame lag is invisible.

Verified in-game on the 4 ai.bucket workers: panel shows the correct
cadences (4/2/1/0.5 hz); halt froze bucket0 (state->paused, iters stop) and
run resumed it; >> took bucket0 to 16 hz; kill marked bucket3 dead with its
buttons greyed.
